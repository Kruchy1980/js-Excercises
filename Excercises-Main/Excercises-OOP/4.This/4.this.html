What is 'this' ;)??


In Simply words, this is a connection with some objects. This let's us to callback to the object. JS according to results assigned to this the connection with object. What the this is exactly the source decides. Assignement of the object to 'this' appears always in the moment of function callback - defined the character is not public. 'This' creates a context (object to which we can callaback inside the code).


Situations when we can meet with this


            1. 'this' in global scope - in case of runtime environment which is a web browser the global scope is equal to /window.
            2. 'this' in function - you will use it mainly. This is created when is called back and creates the connection only than.

Strict mode.


Strict mode was created in JS ES5 for making the code more friendly. The strict mode we can define for only function or for whole file. The most recommended if we want to use it we should use it globally for all the file by inserting on top of the file:

            'use strict';

The strict mode won't let us to defined attribution /przypisaÄ‡ the global object to this when it is under creation. Since the ES6 Class elements contain themselves the strict mode what makes the coding easier so from ES6 we do not need to use the strict mode because it is implemented automatically in a critical situations.

Why 'this' is needed??


'This' gives us no need of using precison functions(separate for each object).

            1.It let's us to use the same function multiple.
            2. Code is cleared and easier to being edited  eg  

            function
            function showName() {
                this.name
            }

            Object 1
            const kate = {
                name: 'Kate',
                showName: showName,                
            }
            Object 2
            const michael = {
                name: 'Michael',
                showName: showName,                
            }
            Check the name of objects
            kate.showName(); //result display name of Object 1
            michael.showName(); //result display name of Object 2

So we can see that once declared function showName is used to each of object thanks to 'this' word.


When the connection with this arises??


The connection of this arises in the moment of executing method and is not dependend of place of function declaration. In a moment of function execution the execution context appears and is a part of 'this'.


Summary now


The source of arising 'this' is the call back of function/method. The connection rises undependendly of if in the function are we using this(as similar as with using event, and we can but no need to use it)
