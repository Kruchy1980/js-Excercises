Abstraction / Abstrakcja - explanation

Abstraction is an implementing tasks specified by the project, a simplified model of reality based on objects. These objects have a specific structure, interface and implementations, purpose, processes and, what is very important, having relationships with other objects.

Relations between the objects - examples:

  • Composition object includes other objects. Composition assumes that one object is embeded in other one. Wherein embeded object is dependend of object which exixts in and it;s existense outside the object it is embeded in does jhave no sense.

  • Association Every object of that relation exixts independently. More of that there exixts relation (connection) between the objects. The existing of one object is not necessary for existing the other one

  • Aggregation object consists from different objects(aggregates them). The main purpose and sense of aggregate(the main object) existing relies on possesion of objects which it's store (but these objects may exists outside the main one)
